Formato de piezas

Formato

BlueBrick solo carga imágenes GIF que están en una carpeta dentro de la carpeta de piezas (vea Organización de carpetas para más detalles). La imagen GIF puede contener color transparente.

No importa la orientación de la vista superior de la pieza en la imagen, pero para facilitar la integración de tu pieza con LDraw, recomiendo usar la misma orientación que la pieza LDraw. De lo contrario tendrás que modificar el archivo LDraw Remap file.

Escala

La escala de una pieza es 1 stud = 8 pixeles. Como en realidad un stud iguala 8mm, otra manera de memorizar esta escala es 1 pixel = 1 milímetro.

Caracteres autorizados para nombrar archivos

Los nombres de las piezas no son sensibles a mayúsculas, ya que BlueBrick convierte todos los nombres a mayúsculas al crear la biblioteca de piezas. Por lo tanto, si creas dos piezas con el mismo nombre, pero alguna letra en minúscula o mayúscula, se considerará como la misma pieza. Puedes usar cualquier carácter autorizado para nombrar tu pieza (incluido espacios), pero recomiendo enérgicamente usar solo caracteres ingleses para facilitar el compartir tu mapa de manera internacional. Evita también el uso del punto ya que este se usa para separar el nombre de un archivo de su extensión y BlueBrick lo emplea para separar el número de una pieza de su color.

Convención de nombres de archivo

A BlueBrick le d igual cómo llamas tus piezas. La única limitación es que el nombre debe ser único dentro de la biblioteca entera. No puedes usar el mismo nombre para dos piezas, incluso si están en carpetas diferentes. Este nombre se convierte en el ID exclusivo que se guarda en el trazado BBM. Para evitar conflictos en la biblioteca y para hacer los archivos BBM portables, habitualmente se emplea el numero de la pieza o set como su nombre. Esto simplemente es un uso común que recomiendo enérgicamente que sigas. La biblioteca oficial de BlueBrick también seguirá este criterio.

Convención de nombres para sets.

Como algunas piezas de Lego© tienen el mismo número que sets de Lego© la npráctica común es la de añadir ".set" como sufijo al número del set. No uses un sufijo numérico (como ".1") ya que ese sufijo se interpretará como un color (vea más abajo para más detalles), y la información del color no es relevante para un set. Además, Peeron y Bricklink pueden emplear un guión seguido de un número para diferenciar piezas de sets y también diferentes sets producidos a lo largo del tiempo pueden tener el mismo número. Por ello es recomendable usar este sufijo y emplaer los mismos números que Peeron o Bricklink.

<NúmeroDeSet>.set.gif
<NúmeroDeSet>.set.xml

Por ejemplo la convención para nombrar el set #2150 es:
2150-1.set.gif

Además, si quieres separar un set en diferentes imágenes gif, puedes renombrárlos de la siguiente manera:
6399-1.set1.gif
6399-1.set2.gif
6399-1.set3.gif
etc...

Si quieres hacer GIFs de modelos alternativos de LEGO que están contenidos dentro de un set, la convención para nombrarlos es como sigue:
4886-1.setA.gif
4886-1.setB.gif
4886-1.setC.gif
etc...

Convención de nombres de piezas

El estándar común para nombrar una pieza de LEGO© es la de usar el número completo de la pieza seguido de un punto, luego el código de color según la tabla de colores de LDraw y finalmente una extensión.

<NúmeroDePieza>.<Color>.gif
<NúmeroDePieza>.<Color>.xml

Por ejemplo, la convención para nombrar un baseplate azúl 32x32 es:
3811.1.gif

Convención de nombres de grupos

El nombre del archivo que describe un grupo de piezas (piezas hechas de piezas organizadas de una manera específica), debería comenzar por un nombre que describe el grupo, seguido de un punto, la palabra "group" y finalmente la extensión. Piezas de grupo nunca tienen imagen gif, solo archivo xml. Si quieres hacer un grupo que representa un set (por ejemplo un set que incluye varios edificios en un formato predefinido), emplea la convención para nombres de sets (vea arriba).

<GroupName>.group.xml

Por ejemplo, la convención de nombre para una vái flexible podría ser:
flex.group.xml
