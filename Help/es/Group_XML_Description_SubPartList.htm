Listar todas las piezas en un grupo

Sintaxis: <SubPartList>
    <SubPart id="...">
        <position>
            <x>...</x>
            <y>...</y>
        </position>
        <angle>...</angle>
    </SubPart>
    ...
</SubPartList>
Valor por defecto: Si esta etiqueta no está presente, querrá decir que el grupo está vacío y será ignorado durante la carga de la biblioteca de piezas.
Descripción: Una lista de piezas con sus posiciones y orientaciones.

El campo SubPartList contiene una lista de elementos SubPart, uno por cada pieza que el grupo contiene. El orden de las piezas en la lista se emplea para determinar el orden en el que se mostrarán superpuestos en pantalla: el primer elemento estará al fondo y el último al frente.

El campo SubPart describe una pieza o sub-grupo dentro de este grupo. El atributo obligatorio id es el número completo de la pieza o el nombre del grupo (sin la extensión .GIF o .XML). Este ID trata mayúsculas y minúsculas como iguales. Si una pieza o grupo no está en la biblioteca, se mostrará una cruz roja en su lugar. Las referencias cíclicas no estás permitidas, lo que significa que no puedes poner una referencia al nombre del grupo en el atributo de ID, ni otro grupo que referencia a este grupo en uno de sus atributos ID. BlueBrick detecta estas referencias cíclicas y generará un error al cargar el archivo y el grupo será ignorado por BlueBrick.

El campo position describe la posición en coordenadas de studs de un elemento dentro del grupo. El origen no tiene importancia, ya que solo se considera la posición relativa entre todos los elementos del grupo. Sin embargo, para facilitar la edición del archivo XML, es conveniente colocar una pieza en el origen (0,0, 0,0) y las demás alrededor. O puedes elegir colocar el origen en el centro del grupo.

El campo angle describe la orientación en grados del elemento, contando los valores positivos en el sentido del reloj..
