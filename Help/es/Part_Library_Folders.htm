Carpetas de la biblioteca de piezas

Organización de las carpetas

La biblioteca de piezas consiste de imágenes en formato GIF y archivos XML, todos guardados dentro una subcarpeta llamada "parts" en la carpeta de instalación de Bluebrick.

 

            ? Esta es la carpeta de la biblioteca de piezas.

Esta carpeta se analiza al inicair el programa y es necesaria para que BlueBrick funcione. Puedes creas tantas subcarpetas dentro de la carpeta "parts" como quieras, pero BlueBrick solo soporta un nivel de profundidad. Piezas colocadas en carpetas dentro de una subcarpeta serán ignoradas. Piezas colocadas directamente dentro de la carpeta "parts" también serán ignoradas.

La estructura orginal de la carpeta "parts" es:
parts
  +- Baseplate
  +- Custom
  +- Duplo
  +- Logo
  +- Rail
  +- Road
  +- Set

Análisis de la carpeta de BlueBrick

Para cada subcarpeta dentro de la carpeta "parts", BlueBrick creará una pestaña en el panel de piezas. El nombre de esta pestaña será el mismo que el de la carpeta. Puedes elegir cualquier nombre de carpeta e incluir espacios y caracteres con acento. Tmbién puedes renombrar, borrar y reorganizar las carpetas originales a tu gusto ya que no se usan para identificar las piezas. Las carpetas simplemente son contenedores convenientes para agrupar las piezas en diferentes pestañas en la biblioteca de piezas. La desventaja es que cada nombre de pieza en toda la biblioteca debe ser único, y el nombre de la pieza es el identificador que se guarda en el archivo de mapa. Normalmente esto no es un problema importante ya que cada pieza de LEGO© ya tiene un número único.

Al crear la biblioteca de piezas, BlueBrick primero busca los archivos GIF en las carpetas de piezas. Para cada GIF que encuentra, BlueBrick comprueba si hay un archivo XML del mismo nombre. Los archivos XML son opcionales y si falta un archivo XML se usarán los valores por defecto para esta pieza.

Luego BluBriick busca archivos XML sin archivo GIF asociado y también cargará estos archivos. Estas piezas pueden ser de dos tipos: bien e un grupo de piezas o una pieza normal. Si se trata de un grupo de piezas, el grupo se recreará en BlueBrick (por eso no tiene un archivo GIF asociado) y será visible en el área de piezas. Si se trata de una pieza normal, esta será ignorada por BluBrick, lo que significa que no aparecerá en el área de piezas. Estas piezas tampoco se verán en un trazado y no generarán un aviso si BluBrick lo encuenytra en un archivo de trazado. De modo que si quiere obligar BlueBrick a ignorar algunas piezas específicas, solo tienes que crear e archivo XML correspondiente.
