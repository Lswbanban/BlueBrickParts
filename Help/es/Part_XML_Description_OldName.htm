Seguir la pista del nombre anterior de luna pieza

Sintaxis: <OldNameList>
    <OldName>...</OldName>
    ...
</OldNameList>
Valor por defecto: Por defecto la etiqueta <OldNameList> no debería aparecer, lo que significa que la pieza nunca fue renombrada.
Descripción: Esta etiqueta solo es de utilidad cuando se renombra una pieza (camba el nombre del archivo GIF y el XML) para todavía poder cargar archivos BBM que se guardaron con el nombre anterior.

Como los números de las piezas que se incluyen en un trazado se listan y guardan en los archivos BBM (archivo del trazado), BlueBrick no encontrará las piezas renombradas y las mostrará como una cruz roja. Para evitar este problema hay que incluir el campo OldNameList en el archivo XML.

Como una pieza puede ser renombrada varias veces, se pueden incluir todos los nombres anteriores de la pieza dentro del campo OldNameList. Este campo contiene una lista de campos OldName cuyo contenido es el numero BlueBrick anterior completo (el numero de la pieza seguido de un punto y el número del color.

Por supuesto solo es necesario añadir este campo si esa pieza fue publicada en Internet con anterioridad. Esto es importante ya que habrá gente que haya creado trazados con el nombre anterior de la pieza. Para facilitarles las cosas, deberías incluir el campo OldNameList para que puedan cargar sus trazados anteriores sin perder imágenes. Si solo cambias el nombre para uso personal, no es necesario guardar un historial completo de los nombres de la pieza.

Nota que si cargas un archivo de trazado con un nombre viejo y lo guardas nuevamente, se usará el nuevo nombre al guardar. Sin embargo, jamas debes borrar un campo OldNameList a menos que estés seguro de que todos los archivos de trazado que usaban el nombre antiguo han sido actualizados. Como este campo no el rendimiento de BlueBrick puedes dejarlo, por si acaso.

También puedes usar el campo OldNameList como un alias para una pieza porque de hecho BlueBrick trata los nombres antiguos como alias. Sin embargo, si añades una pieza nueva a la biblioteca que tiene el mismo nombre que una pieza anterior que ha sido renombrada, el alias de la nueva pieza será ignorado. Esto significa que BlueBrick solo crea el alias si no hay otra pieza con el mismo nombre.
